# 株式会社ラボル｜フロントエンドエンジニア スキルテスト 事前案内

## 概要

このプロジェクトは、株式会社ラボルのフロントエンドエンジニア採用試験で使用します。

試験当日に弊社人事よりメールで問題を配布します。内容に従ってコーディングを行ってください。

試験時間は最長60分です。早く完了した場合は、その時点で提出していただいて構いません。
なお、提出時間の速さは合否に影響しません。

## 試験の流れ

1. 事前にこのREADME.mdをご確認ください
2. 試験当日、開始時刻の10分前に弊社人事がメール※から問題を配布します
3. 開始時刻になったら回答を開始してください
4. 回答終了後、15分以内（提出締め切り時刻はメールに記載）に「回答の提出手順」に従って提出してください
5. 提出が完了したら、メール※に「解答終了し、提出しました」とご返信ください
6. 5が完了したら試験終了です。選考結果は後日ご連絡します

※ メールの件名：【フロントエンドエンジニア スキルテスト：本番案内】株式会社ラボル

### 回答の提出手順

1. 回答ファイルをフォルダにまとめ、zip形式で圧縮してください（フォルダ名は `回答` としてください）
2. 事前案内メールに記載のログインURLからAWS管理コンソールにログインします（認証情報は同メール内に記載）
3. 事前案内メールに記載の回答用フォルダURLから提出フォルダを開きます
4. 画面中央のアップロードボタンを押して、手順1でzip化した回答フォルダをアップロードしてください

## 注意事項

- Web検索の使用は問題ありません
- 代理回答は不可です
- フレームワークは使用不可です（問題はベースコードの修正となります）
- 提出前に、ブラウザで正常に表示できるか確認してください（推奨ブラウザ：Safari / Chrome / Firefox / Edge の最新版）
- zip以外の形式で提出された場合は採点対象外となります

## このプロジェクトのコードについて

試験当日に配布される問題に取り組むためのベースコード（HTML / CSS / JS）です。
商品一覧を表示する簡易アプリケーションです。

### ファイル構成

- `index.html`
- `style.css`
- `app.js`

### 重要な概念

#### 状態管理

状態は `state` オブジェクトに集約しています。

```javascript
const state = {
  products: [],                // 商品データ
  selectedProductType: 'ALL'   // 選択中の商品タイプ
};
```

状態を変更したら、必ず `updateView()` を呼んで画面を更新します。

#### 画面更新の流れ

1. イベントが発生する（ボタンクリック、セレクト変更など）
2. イベントハンドラが `state` を更新する
3. `updateView()` を呼んで画面全体を再描画する

この流れを守ることで、状態と表示が常に一致します。

#### 主要な関数

- `loadProducts()` - 商品データを取得して `state.products` に格納する
- `updateView()` - `state` の内容に基づいて画面全体を再描画する。状態が変わったら必ず呼ぶ
- `getFilteredProducts()` - 選択中の商品タイプでフィルタリングした商品配列を返す
- `createProductCard(product)` - 商品1件分のHTMLを生成して返す

### データ構造

#### 商品オブジェクトの形式

```javascript
{
  productId: 1,               // 商品ID
  name: "ラーメン",            // 商品名
  productType: "FOOD",        // 商品タイプ（FOOD または DRINK）
  price: 720,                 // 価格（円）
  image: "https://..."        // 画像URL
}
```

#### 商品タイプ

- `FOOD` - 食品
- `DRINK` - 飲料

### ローカルでの動作確認

サーバーは不要です。`index.html` をブラウザで直接開くと動作します。
